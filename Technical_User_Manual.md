# מדריך משתמש ומסמך טכני - CompareD

מסמך זה משמש כמדריך טכני ומדריך משתמש מפורט עבור מערכת השוואת הנתונים הדינמית CompareD. המדריך מפרט את זרימת העבודה של המשתמש בממשק מול הפעולות הטכניות המתבצעות מאחורי הקלעים בבקר השרת.

## 1. מסך הגדרת חיבורי מסדי נתונים (דף הבית)

דף הבית הוא נקודת הכניסה למערכת שבה המשתמש מגדיר את פרטי החיבור של שתי סביבות העבודה.

### מדריך למשתמש:
1. מה רואים במסך: שתי תיבות טקסט רחבות בתוך פריסת Bento Grid מעוצבת. התיבה הימנית מיועדת עבור Legacy SQL Server והתיבה השמאלית עבור New Oracle Database.
2. מה מזינים: מחרוזת חיבור (Connection String) חוקית עבור כל אחד ממסדי הנתונים. תיבות הטקסט כוללות דוגמאות מובנות למבנה מחרוזת החיבור.
3. היכן לוחצים: לוחצים על הכפתור הזוהר "התחבר והבא טבלאות" שנמצא בתחתית הדף.
4. ניהול שגיאות: במידה והחיבור לאחד ממסדי הנתונים נכשל, הדף ייטען מחדש ויציג תיבת שגיאה אדומה וברורה בעיצוב Pro Max המפרטת את סיבת השגיאה, תוך שמירה על מחרוזות החיבור שהוזנו בתוך תיבות הטקסט.

### מדריך טכני למפתח:
1. בקר ופעולה: הבקשה נשלחת כשיטת POST אל הפעולה Connect בבקר CompareController.cs.
2. העברת נתונים: מחרוזות החיבור מועברות כפרמטרים sqlConnectionString ו-oracleConnectionString מהטופס.
3. פעילות מאחורי הקלעים: הבקר בודק תקינות בסיסית של המחרוזות ומפעיל במקביל (באמצעות Task.WhenAll) שתי משימות אסינכרוניות. המשימה הראשונה פונה ל-SQL Server באמצעות SqlConnection ומריצה שאילתה מובטחת השולפת את שמות הטבלאות והתצוגות (Views) הזמינות. המשימה השנייה פונה ל-Oracle באמצעות OracleConnection ומריצה שאילתה מובטחת המאחדת טבלאות ותצוגות השייכות למשתמש הנוכחי.
4. ניהול סשן ומעבר שלב: אם משימות החיבור הצליחו, מחרוזות החיבור נשמרות בבטחה בסשן השרת (HttpContext.Session) תחת המפתחות SqlConnectionString ו-OracleConnectionString. הבקר יוצר מודל מסוג TableSelectionViewModel המכיל את רשימות האובייקטים ומפנה את המשתמש לתצוגה SelectTables.cshtml. אם החיבור נכשל, הודעת השגיאה ומחרוזות החיבור נשמרות ב-TempData והבקר מבצע הפניה מחדש (RedirectToAction) אל פעולת Index בבקר הבית.

## 2. מסך בחירת טבלאות ותצוגות להשוואה

מסך זה מאפשר למשתמש לבחור אילו טבלאות או תצוגות (Views) הוא מעוניין להשוות בין שתי המערכות.

### מדריך למשתמש:
1. מה רואים במסך: שתי תיבות בחירה (Dropdowns) המציגות את רשימת הטבלאות והתצוגות שנשלפו. מעל כל תיבת בחירה מופיעה תיבת טקסט לחיפוש מהיר.
2. מה מזינים: המשתמש מקליד בתיבת החיפוש המהיר כדי לסנן באופן מיידי את רשימת האובייקטים לפי השם. לאחר מכן המשתמש בוחר טבלה או תצוגה מתיבת הבחירה עבור SQL Server (מקור) ועבור Oracle (יעד).
3. היכן לוחצים: לוחצים על הכפתור הזוהר "המשך למיפוי שדות" שנמצא בתחתית הדף.

### מדריך טכני למפתח:
1. סינון בצד הלקוח: קוד JavaScript המוטמע בתצוגה שומר את רשימת האופציות המקורית ומסנן אותן דינמית על בסיס הזנת המשתמש בתיבת החיפוש. הסינון מבוצע בצד הלקוח ללא פנייה לשרת לטובת חוויית משתמש מהירה.
2. בקר ופעולה: שליחת הטופס מפעילה את הפעולה SelectFields בבקר CompareController.cs כשיטת POST.
3. העברת נתונים: שמות הטבלאות שנבחרו מועברים כפרמטרים sqlTable ו-oracleTable מהטופס.
4. פעילות מאחורי הקלעים: הבקר שולף את מחרוזות החיבור השמורות מהסשן ומבצע במקביל שתי שאילתות Parameterized מאובטחות (למניעת הזרקת SQL). השאילתה הראשונה שולפת את שמות העמודות של הטבלה שנבחרה ב-SQL Server מתוך INFORMATION_SCHEMA.COLUMNS. השאילתה השנייה שולפת את שמות העמודות ב-Oracle מתוך USER_TAB_COLUMNS.
5. מעבר שלב: הבקר שומר את שמות הטבלאות הנבחרות בסשן השרת, יוצר מודל מסוג FieldMappingViewModel המכיל את שמות העמודות שחזרו, ומחזיר את תצוגת מיפוי השדות MapFields.cshtml.

## 3. מסך הגדרת מיפוי שדות

במסך זה המשתמש קובע כיצד להשוות את השדות בין הטבלאות שנבחרו.

### מדריך למשתמש:
1. מה רואים במסך: שני כרטיסי בחירה רחבים המציגים את מצב המיפוי (השוואה אוטומטית מלאה מול מיפוי ידני).
2. בחירת אפשרויות: אם בוחרים באפשרות א (השוואה אוטומטית מלאה), המערכת תבצע השוואה אוטומטית ללא הגדרה נוספת. אם בוחרים באפשרות ב (מיפוי ידני מותאם אישית), ייפתח בונה שורות דינמי שבו ניתן להוסיף שורות מיפוי.
3. הזנה במצב ידני: עבור כל שורה שהתווספה, המשתמש בוחר שדה מקור ב-SQL Server, שדה יעד תואם ב-Oracle ואת תפקיד השדה (שדה מפתח שעל בסיסו מתאימים את השורות, או שדה להשוואה שאת תוכנו בודקים).
4. הגבלת כמות רשומות: בתחתית המיפוי מופיעה תיבת קלט מספרית בשם "הגבלת כמות רשומות לטעינה" עם ערך ברירת מחדל של 1000. המשתמש יכול לשנות מספר זה כדי לשלוט בכמות הרשומות המקסימלית שתיטען מכל מסד נתונים.
5. היכן לוחצים: לוחצים על הכפתור "הרץ השוואת נתונים" כדי להתחיל את תהליך ההשוואה.

### מדריך טכני למפתח:
1. אינטראקציה בצד הלקוח: קוד JavaScript מנהל את המעבר בין הכרטיסים ומציג/מסתיר את אזור המיפוי הידני על ידי שינוי ערך השדה המוסתר mappingMode. פונקציות הוספה ומחיקה מנהלות את יצירת שורות המיפוי בתוך הטבלה בצורה דינמית.
2. בקר ופעולה: שליחת הטופס מפעילה כשיטת POST את הפעולה CompareData בבקר CompareController.cs.
3. העברת נתונים: נתוני הטופס מועברים כפרמטרים הבאים: sqlTable (שם טבלת SQL), oracleTable (שם טבלת Oracle), mappingMode (מצב המיפוי), sourceFields (רשימת שדות המקור שנבחרו במצב ידני), targetFields (רשימת שדות היעד שנבחרו במצב ידני), fieldRoles (רשימת תפקידי השדות המתאימים לשורות) ו-maxRows (כמות הרשומות המקסימלית לטעינה).
4. פעילות מאחורי הקלעים: הבקר בודק את מצב המיפוי. במצב אוטומטי, הבקר מבצע שליפה מקדימה של העמודות, מוצא עמודות בעלות שם זהה, מגדיר את העמודה הראשונה כמפתח והשאר להשוואה. במצב ידני, הבקר מחלק את השדות שהועברו לפי תפקידם (מפתח מול שדה להשוואה).
5. בניית שאילתות דינמיות מאובטחות: הבקר מרכיב שאילתות SELECT השולפות אך ורק את העמודות המשתתפות בתהליך (למניעת עומס נתונים). השאילתה ל-SQL Server משתמשת במגבלת TOP (maxRows) והשאילתה ל-Oracle משתמשת במגבלת FETCH FIRST maxRows ROWS ONLY בסוף השאילתה.
6. ביצוע ההשוואה בזיכרון: הבקר פותח את שני החיבורים, שולף את הרשומות במקביל ומאחסן אותן בשני מילונים בזיכרון השרת. המפתח של כל מילון מורכב מערכי שדות המפתח המחוברים בתו המפריד | (Composite Key).
7. לוגיקת ההשוואה: המערכת עוברת על רשומות המקור ומחפשת התאמה ביעד. רשומות עם ערכים שונים מסומנות כבעלות הבדלים (Difference). רשומות הקיימות רק במקור מסומנות כחסרות ביעד (MissingInOracle). לאחר מכן, המערכת עוברת על רשומות היעד ומאתרת רשומות שאינן קיימות כלל במקור (MissingInSql).
8. מעבר שלב: הבקר יוצר מודל מסוג ComparisonResultViewModel המכיל את כל נתוני הסיכום והפירוט, ומפנה לתצוגת התוצאות Results.cshtml.

## 4. מסך תוצאות ההשוואה

מסך זה מציג למשתמש את הדוח המסכם של ממצאי ההשוואה בצורה ויזואלית ונקייה.

### מדריך למשתמש:
1. מה רואים במסך: בראש המסך מוצג דשבורד מסכם בפריסת Bento Grid עם ארבעה כרטיסי סיכום: רשומות זהות (בירוק זוהר), הבדלי נתונים (באדום זוהר), רשומות חסרות ביעד (בצהוב זוהר) ורשומות חסרות במקור (בכחול זוהר).
2. טבלת פירוט: מתחת לדשבורד מוצגת טבלה המפרטת את כל השורות הלא תואמות (שורות עם הבדלי נתונים או שורות חסרות). עבור שורות עם הבדלים, מוצגת רשימת השדות המושווים, כאשר שדה תקין נצבע בירוק עדין ושדה לא תואם נצבע באדום עדין ומראה את הערך במקור מול הערך ביעד.
3. היכן לוחצים: לוחצים על הכפתור "התחל השוואה חדשה" כדי לחזור לדף הבית ולהתחיל תהליך השוואה חדש.

### מדריך טכני למפתח:
1. רינדור המידע: התצוגה Results.cshtml מקבלת את מודל הנתונים ומבצעת לולאת רינדור על רשימת הפירוט (Model.Details).
2. עיצוב מבוסס תנאים: שימוש בתנאי Razor מאפשר לבדוק את סטטוס השורה ואת תקינות כל שדה (field.IsMatch) ולהחיל סגנונות רקע וגבול מותאמים אישית (אדום או ירוק שקופים חלקית) המבטיחים ניגודיות נאותה ומראה Pro Max.
3. חזרה להתחלה: קישור הקיים בכפתור ההתחלה מנווט מחדש אל HomeController.Index ומאפשר ריקון סשן או תחילת עבודה חדשה.
