# פרויקט CompareD - מערכת להשוואת נתונים דינמית

מערכת אינטרנטית מודרנית המיועדת להשוואת נתונים מהירה, מאובטחת ודינמית בין מסד נתונים ישן (SQL Server) למסד נתונים חדש (Oracle) לצורך אימות ואישור תהליך מיגרציה. המערכת מבוססת על טכנולוגיית ASP.NET Core MVC ומשתמשת בגישה ישירה ומאובטחת דרך ADO.NET נקי ללא שימוש ב-Entity Framework, המאפשר ביצועים מיטביים וניידות קלה בין מחשבי עבודה.

## ארכיטקטורה וזרימת העבודה במערכת

אפליקציית CompareD פועלת לפי ארכיטקטורת MVC (מודל, תצוגה ובקר) מסורתית אך מעוצבת לפי עקרונות ממשק מודרניים:

1. אתחול האפליקציה: שרת האינטרנט מופעל דרך קובץ Program.cs המגדיר את שירותי ה-MVC, שירותי המטמון בזיכרון (Memory Cache) והפעלת מצב Session לאחסון נתוני החיבור.
2. ניתוב הבקשות: הבקר (CompareController) מקבל את הבקשות מבוססות ה-HTTP, פונה למסדי הנתונים במידת הצורך ומחזיר את מודל הנתונים המתאים לתצוגה הרלוונטית.
3. הצגת הממשק (View): תצוגות ה-Razor (.cshtml) מרנדרות את ממשק המשתמש בצד הלקוח על בסיס עיצוב כהה עמוק (Deep Dark Mode), הכולל אפקטים של Aurora UI ו-Glassmorphism להשגת מראה יוקרתי ומקצועי.

## ניהול מאובטח של מחרוזות החיבור בסשן

אבטחת המידע וניידות הפרויקט מהווים נדבך מרכזי במערכת:

1. הזנת הנתונים: המשתמש מזין את מחרוזות החיבור בשדות הקלט הייעודיים בדף הבית.
2. אימות ושמירה: לאחר ביצוע בדיקת התחברות מוצלחת במקביל לשני מסדי הנתונים, מחרוזות החיבור נשמרות בתוך ה-HttpContext.Session בשרת.
3. הגנה על נתוני החיבור: נתוני החיבור אינם נשמרים במסד נתונים מקומי ואינם מועברים בשדות נסתרים בדפי ה-HTML, ובכך נמנעת חשיפתם לצד הלקוח.
4. שימוש חוזר: בשלבי בחירת הטבלאות, שליפת העמודות והרצת ההשוואה, הבקר שולף את מחרוזות החיבור ישירות מהסשן המאובטח של השרת ללא מעורבות הלקוח.

## שלבי זרימת העבודה באפליקציה

המערכת פועלת בצורה מובנית דרך ארבעה שלבים ברורים:

1. שלב החיבור (Connection Phase): הזנת מחרוזות החיבור של SQL Server ושל Oracle ובדיקת קישוריות בו-זמנית.
2. שלב בחירת האובייקטים (Selection Phase): הצגת רשימת הטבלאות והתצוגות (Views) הזמינות בכל אחד מהמסדים עם אפשרות חיפוש וסינון מהיר מבוסס JavaScript בצד הלקוח.
3. שלב מיפוי השדות (Mapping Phase): בחירה בין השוואה אוטומטית מלאה לבין מיפוי ידני מותאם אישית שבו המשתמש מגדיר את שדות המפתח (Key Fields) ואת השדות להשוואה (Value Fields).
4. שלב ההרצה והצגת התוצאות (Execution Phase): טעינת הנתונים למבני נתונים בזיכרון, ביצוע ההשוואה והצגת דשבורד מסכם הכולל חיווי ברור של אי-התאמות.

## פירוט אלגוריתם ההשוואה

אלגוריתם ההשוואה פועל בזיכרון השרת ומתוכנן לביצועים מרביים:

1. שליפת עמודות: הבקר בונה שאילתות Parameterized מאובטחות (למניעת הזרקת קוד) השולפות רק את העמודות המבוקשות (לפי המיפוי) מטבלת המקור ומטבלת היעד. השאילתות מגבילות את כמות הרשומות הנטענות באופן דינמי לפי ערך ה-Max Rows שהמשתמש קבע בממשק (באמצעות שימוש ב-TOP ב-SQL Server וב-FETCH FIRST ב-Oracle).
2. טעינת הנתונים: הרשומות נטענות במקביל לתוך שני מילונים (Dictionaries) בזיכרון. המפתח של כל מילון מיוצג כמפתח מורכב (Composite Key) המבוסס על שילוב הערכים של שדות המפתח שנבחרו. ערך המילון הוא מילון פנימי המכיל את שמות העמודות וערכיהן.
3. השוואת מקור מול יעד: המערכת עוברת על רשומות SQL Server ומחפשת התאמה במילון של Oracle לפי מפתח השורה. אם נמצאה רשומה תואמת, מתבצעת השוואה של כל אחד משדות הערך. במידה וקיים שדה שאינו זהה (ללא הבדל רווחים או רישיות), השורה מסומנת כבעלת הבדלים (Difference). אם המפתח אינו קיים כלל ב-Oracle, השורה מסומנת כחסרה ביעד (MissingInOracle).
4. איתור רשומות חסרות במקור: המערכת עוברת על רשומות Oracle ובודקת אילו מפתחות אינם קיימים כלל במילון של SQL Server. שורות אלו מסומנות כחסרות במקור (MissingInSql).
5. רינדור התוצאות: תוצאות ההשוואה נשלחות לתצוגת Results.cshtml המציגה דשבורד מסכם (בנטו גריד) וטבלת הבדלים המדגישה תאים שונים בצבע אדום עדין ותאים זהים בצבע ירוק עדין.
