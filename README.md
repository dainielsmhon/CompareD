# פרויקט CompareD - מערכת להשוואת נתונים דינמית

מערכת אינטרנטית מודרנית המיועדת להשוואת נתונים מהירה, מאובטחת ודינמית בין מסד נתונים ישן (SQL Server) למסד נתונים חדש (Oracle) לצורך אימות ואישור תהליך מיגרציה. בנוסף, המערכת תומכת בהשוואה ישירה של קובצי נתונים עצמאיים (CSV) ללא תלות בחיבור פעיל למסדי נתונים.
המערכת מבוססת על טכנולוגיית ASP.NET Core MVC ומשתמשת בגישה ישירה ומאובטחת דרך ADO.NET נקי ללא שימוש ב-Entity Framework, המאפשר ביצועים מיטביים וניידות קלה בין מחשבי עבודה.

---

## תכונות מרכזיות במערכת

1. **השוואת מסדי נתונים (SQL Server vs Oracle)**: השוואה חכמה בזמן אמת של טבלאות/תצוגות לפי מפתח ראשי והגדרה מותאמת אישית של שדות להשוואה.
2. **השוואת קובצי CSV**: אפשרות להעלאת שני קובצי CSV (מקור ויעד) וביצוע השוואה אוטומטית בזיכרון, כולל זיהוי מפתח ראשי ומיפוי עמודות זהות.
3. **נתוני דמי מובנים (Mock Data)**: תמיכה בפרופיל חיבור מדומה (`MockConnectionString`) להדגמה ובדיקה מהירה של מסכי המערכת ללא תלות במסדי נתונים פיזיים.
4. **ייצוא דוחות מתקדם**: כפתורי הורדה בדשבורד התוצאות המאפשרים ייצוא של ממצאי ההשוואה לקובצי Excel (CSV עם UTF-8 BOM לתמיכה בעברית) ולקובצי Word (HTML-Doc מעוצב).
5. **ייבוא וייצוא תצורת מיפוי (JSON)**: שמירה של הגדרות מיפוי שדות מורכבות לקובץ JSON וטעינתן מחדש לקיצור תהליכי השוואה עתידיים.
6. **ממשק משתמש Pro Max**: עיצוב כהה עמוק (Deep Dark Mode) מרהיב, הכולל אפקטים של Aurora UI, זכוכית מטושטשת (Glassmorphism), כפתורים זוהרים וסינון מהיר בזמן אמת.

---

## ארכיטקטורה וזרימת העבודה במערכת

אפליקציית CompareD פועלת לפי ארכיטקטורת MVC (מודל, תצוגה ובקר) מסורתית אך מעוצבת לפי עקרונות ממשק מודרניים:

1. **אתחול האפליקציה**: שרת האינטרנט מופעל דרך קובץ `Program.cs` המגדיר את שירותי ה-MVC, שירותי המטמון בזיכרון (Memory Cache) והפעלת מצב Session לאחסון נתוני החיבור.
2. **ניתוב הבקשות**: הבקר (`CompareController`) מקבל את הבקשות מבוססות ה-HTTP, פונה למסדי הנתונים במידת הצורך או מפרש קובצי קלט, ומחזיר את מודל הנתונים המתאים לתצוגה הרלוונטית.
3. **הצגת הממשק (View)**: תצוגות ה-Razor (.cshtml) מרנדרות את ממשק המשתמש בצד הלקוח על בסיס עיצוב כהה עמוק, המבטיח חוויית שימוש מתקדמת.

---

## ניהול מאובטח של מחרוזות החיבור בסשן

אבטחת המידע וניידות הפרויקט מהווים נדבך מרכזי במערכת:

1. **הזנת הנתונים**: המשתמש מזין או בוחר את פרופילי החיבור בשדות הקלט הייעודיים בדף הבית.
2. **אימות ושמירה**: לאחר ביצוע בדיקת התחברות מוצלחת במקביל לשני מסדי הנתונים, מחרוזות החיבור נשמרות בתוך ה-`HttpContext.Session` בשרת.
3. **הגנה על נתוני החיבור**: נתוני החיבור אינם נשמרים במסד נתונים מקומי ואינם מועברים בשדות נסתרים בדפי ה-HTML, ובכך נמנעת חשיפתם לצד הלקוח.
4. **שימוש חוזר**: בשלבי בחירת הטבלאות, שליפת העמודות והרצת ההשוואה, הבקר שולף את מחרוזות החיבור ישירות מהסשן המאובטח של השרת ללא מעורבות הלקוח.

## מסלולי זרימת העבודה באפליקציה

### מסלול א׳: השוואת מסדי נתונים (שישה שלבים מובנים)
1. **שלב 1 - חיבור**: בחירת פרופילי החיבור של SQL Server ושל Oracle ובדיקת קישוריות בו-זמנית.
2. **שלב 2 - בחירת טבלאות**: הצגת רשימת הטבלאות והתצוגות (Views) הזמינות בכל אחד מהמסדים עם אפשרות חיפוש וסינון מהיר מבוסס JavaScript בצד הלקוח.
3. **שלב 3 - סקירת סכמה**: הצגת השוואת סכמה מקיפה (סוגי נתונים, עמודות חסרות) והצעת מפתח ראשי ברירת מחדל.
4. **שלב 4 - מיפוי שדות (בדיקות מקדימות)**: בחירה בין השוואה אוטומטית מלאה לבין מיפוי ידני מותאם אישית שבו המשתמש מגדיר את שדות המפתח (Key Fields) ואת השדות להשוואה (Value Fields).
5. **שלב 5 - השוואה**: טעינת הנתונים למבני נתונים בזיכרון וביצוע אלגוריתם ההשוואה.
6. **שלב 6 - דוח תוצאות**: הצגת דשבורד מסכם הכולל חיווי ברור של אי-התאמות, כפילויות, חוסרים וקיבוץ בעיות לפי תבניות.

### מסלול ב׳: השוואה ישירה של קובצי CSV/Excel
1. **שלב 1 - העלאת קבצים**: העלאת שני קובצי נתונים (CSV) עבור מקור (Table 1) ויעד (Table 2).
2. **שלב 2 - דוח תוצאות**: עיבוד הקבצים בזיכרון, זיהוי כותרות ומיפוי אוטומטי, והצגת דשבורד התוצאות הסטנדרטי באופן מיידי.

## פירוט אלגוריתם ההשוואה

אלגוריתם ההשוואה פועל בזיכרון השרת ומתוכנן לביצועים מרביים:

1. **שליפת עמודות (במסלול מסדי נתונים)**: הבקר בונה שאילתות Parameterized מאובטחת השולפות רק את העמודות המבוקשות (לפי המיפוי) מטבלת המקור ומטבלת היעד. השאילתות מגבילות את כמות הרשומות הנטענות באופן דינמי לפי ערך ה-`Max Rows` שהמשתמש קבע בממשק.
2. **פירוש קבצים (במסלול קבצים)**: מחלקת `CsvParser` קוראת את זרם הנתונים (Stream) של קובצי ה-CSV, מפרקת את השורות תוך טיפול במרכאות כפולות ופסיקים, וממירה אותם למבנה נתונים מסוג `List<Dictionary<string, object>>`.
3. **טעינת הנתונים**: הרשומות נטענות לתוך שני מילונים (Dictionaries) בזיכרון. המפתח של כל מילון מיוצג כמפתח מורכב (Composite Key) המבוסס על שילוב הערכים של שדות המפתח שנבחרו.
4. **השוואת מקור מול יעד**: המערכת עוברת על רשומות המקור ומחפשת התאמה ביעד לפי מפתח השורה.
   - אם נמצאה רשומה תואמת, מתבצעת השוואה של כל אחד משדות הערך (תוך נרמול ערכים ריקים, "NULL", וערכי '0' כזהים). במידה וקיים שדה שאינו זהה, השורה מסומנת כבעלת הבדלים (`Difference`) ומקובצת תחת תבנית הפרש מתאימה.
   - אם המפתח אינו קיים כלל ביעד, השורה מסומנת כחסרה ביעד (`MissingInOracle`).
5. **איתור רשומות חסרות במקור**: המערכת עוברת על רשומות היעד ובודקת אילו מפתחות אינם קיימים כלל במקור. שורות אלו מסומנות כחסרות במקור (`MissingInSql`).
6. **רינדור התוצאות**: תוצאות ההשוואה מועברות לתצוגת `Results.cshtml` המציגה דשבורד מסכם (בנטו גריד) וטבלת הבדלים מפורטת עם חיווי צבעים דינמי.
