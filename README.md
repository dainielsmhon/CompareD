# פרויקט CompareD - מערכת השוואת נתונים דינמית ומאובטחת

מערכת אינטרנטית מודרנית ומאובטחת המיועדת להשוואת נתונים מהירה, דינמית ומוגנת בזמן אמת בין מסד נתונים ישן (**SQL Server**) למסד נתונים חדש (**Oracle**) כחלק מתהליכי מיגרציה, אימות והסבה של נתוני ארגון. בנוסף, המערכת תומכת בהשוואה ישירה של קובצי נתונים עצמאיים (**CSV** או **Excel .xlsx**) ללא תלות בחיבור פעיל למסדי נתונים פיזיים.

המערכת מבוססת על טכנולוגיית **ASP.NET Core MVC** ומשתמשת בגישה ישירה ומאובטחת דרך **ADO.NET** נקי ללא שימוש ב-Entity Framework, מה שמבטיח ביצועים אופטימליים (High Performance) וניידות מלאה וקלה בין סביבות עבודה שונות.

---

## 🚀 תכונות מרכזיות במערכת

1. **השוואת מסדי נתונים (SQL Server vs Oracle)**:
   * התחברות דינמית לשני בסיסי הנתונים במקביל.
   * השוואה חכמה בזמן אמת של טבלאות/תצוגות (Views) על פי מפתח ראשי ייעודי.
   * תמיכה בפרופיל חיבור מדומה (`MockConnectionString`) המאפשר הדגמה ובדיקה מהירה של מסכי המערכת ללא תלות במסדי נתונים פיזיים.

2. **השוואת קובצי CSV ו-Excel (.xlsx)**:
   * העלאה של שני קובצי נתונים (מקור ויעד) וביצוע השוואה מותאמת אישית.
   * קריאה מהירה וחסכונית בזיכרון של קובצי אקסל באמצעות ספריית `MiniExcel` ישירות מזרם הנתונים (Stream) ללא צורך בהתקנת Excel או פתיחה ידנית של ארכיונים.

3. **סקירה, מיפוי והשוואה מותאמת אישית (Dynamic Schema Mapping)**:
   * בחירה וזיהוי של שדות המפתח הראשי (Key Fields) של המקור והיעד.
   * בחירה ומיפוי של עמודות ההשוואה (Value Fields).
   * תמיכה בייבוא וייצוא של תצורת מיפוי השדות לקובץ **JSON** חיצוני לצורך שימוש חוזר עתידי.

4. **דשבורד תוצאות מתקדם וייצוא דוחות**:
   * תצוגה גרפית מסכמת ומעוצבת של ממצאי ההשוואה (התאמות, אי-התאמות, כפילויות וחוסרים).
   * קיבוץ אוטומטי של בעיות לפי תבניות הפרש (Patterns) לזיהוי מהיר של תקלות רוחביות.
   * ייצוא דוחות ל-**Excel** (קובצי CSV מקודדים ב-UTF-8 BOM לתמיכה מלאה בעברית) ול-**Word** (HTML-Doc מעוצב ומסוגנן).

5. **ממשק משתמש Pro Max בעיצוב מודרני**:
   * עיצוב כהה עמוק מרהיב (**Deep Dark Mode**).
   * שילוב של אפקטים מודרניים כגון **Aurora UI**, זכוכית מטושטשת (**Glassmorphism**) ואלמנטים זוהרים להענקת חוויית משתמש פרמיום.
   * סינון וחיפוש מהיר בזמן אמת בצד הלקוח באמצעות JavaScript.

## 🔒 אבטחת מידע והקשחת מערכת (Security Hardening)

המערכת עברה תהליך הקשחה מקיף המותאם לעבודה מאובטחת בסביבות ארגוניות רגישות ובסביבות מנותקות רשת (**Air-Gapped**):

* **אימות מול Active Directory (Windows Authentication)**:
  * הטמעת מנגנון אימות מובנה המבוסס על פרוטוקול `Negotiate` (Kerberos/NTLM) לזיהוי אוטומטי ומאובטח של משתמשי ה-Domain ללא צורך בהקלדת סיסמאות באפליקציה.
* **הגבלת הרשאות ניהול (Role-Based Authorization)**:
  * הגנה על בקר הניהול (`AdminController`) באמצעות אטריביוט האבטחה `[Authorize]` ובדיקה קפדנית של זהות המשתמש אל מול רשימת המנהלים המורשים (`AuthorizedUsers`) המוגדרת בקובץ התצורה.
  * הפניה מאובטחת לדף חסימת גישה ייעודי (Access Denied) במקרה של משתמש לא מורשה או חסום.
* **הגנה מפני התקפות DoS והצפת משאבים**:
  * **הגבלת גודל בקשה**: הגדרת מגבלת נפח קשיחה ברמת שרת ה-Kestrel וה-MVC ל-**105MB** לכל בקשה (`MultipartBodyLengthLimit` ו-`MaxRequestBodySize`).
  * **מגבלת קצב חיבורים (Rate Limiting)**: שימוש ב-[ConnectRateLimiter](file:///c:/Users/danie/CompareD/Services/ConnectRateLimiter.cs) מבוסס Memory Cache להגבלת ניסיונות חיבור חוזרים למסד הנתונים (עד 10 ניסיונות ב-5 דקות) למניעת התקפות Brute-Force.
* **אבטחת תקשורת ודפדפן**:
  * **חסימת הזרקות וסקריפטים**: שילוב [SecurityHeadersMiddleware](file:///c:/Users/danie/CompareD/Middleware/SecurityHeadersMiddleware.cs) המוסיף כותרות אבטחה תקניות לכל תגובה (`Content-Security-Policy`, `X-Content-Type-Options`, `X-Frame-Options`, `Referrer-Policy`).
  * **אבטחת עוגיות (Cookies)**: הגדרת עוגיות ה-Session וה-Antiforgery כ-`HttpOnly = true` ועם מדיניות `SecurePolicy = Always` (מעבר ב-HTTPS בלבד).
* **ארכיטקטורת Read-Only מוחלטת**:
  * הקוד מתוכנן לביצוע שאילתות `SELECT` בלבד. אין במערכת הרשאות או לוגיקה לביצוע שינויים בנתוני הארגון.
  * כל עיבודי הנתונים, הנרמולים והשוואת המפתחות מבוצעים בזיכרון השרת (In-Memory) בלבד, ללא שמירת עקבות או יצירת טבלאות זמניות בבסיסי הנתונים.
* **בקרת פעולות (Audit Log)**:
  * תיעוד מפורט של כניסות משתמשים, ניסיונות גישה לא מורשים, שינויי סטטוס והרצת השוואות בקובץ לוג פנימי בשרת (`logs/audit.log`).

## 🛠 ארכיטקטורה וטכנולוגיות

* **Framework**: .NET 8.0 / .NET 9.0 (ASP.NET Core MVC)
* **Data Access**: ADO.NET (Microsoft.Data.SqlClient, Oracle.ManagedDataAccess.Core) ללא ORM לביצועים מרביים.
* **Excel Parsing**: `MiniExcel` (ביצועים מהירים במיוחד וצריכת זיכרון אפסית).
* **Frontend**: HTML5, Vanilla CSS (Deep Dark Theme), Bootstrap 5.3 (RTL), Vanilla JavaScript.
* **Session & Cache**: `Microsoft.Extensions.Caching.Memory` לניהול Rate Limiting ו-Session מבוסס זיכרון לאחסון סודות חיבור בצורה מאובטחת בשרת.

## 💻 הפעלה והרצה מקומית

### דרישות קדם
* התקנת .NET SDK 8.0 ומעלה.

### שלבי הרצה

1. **הגדרת משתמשים מורשים בניהול**:
   פתחו את הקובץ [appsettings.json](file:///c:/Users/danie/CompareD/appsettings.json) והגדירו את שם משתמש המחשב/Domain שלכם תחת `AuthorizedUsers`:
   ```json
   "AdminSettings": {
     "AuthorizedUsers": [
       "DESKTOP-NAME\\your-username"
     ]
   }
   ```

2. **הרצת האפליקציה**:
   פתחו מסוף (Terminal) בתיקיית הפרויקט והריצו:
   ```bash
   dotnet run
   ```

3. **גישה למערכת**:
   לאחר הפעלת השרת, פתחו את הדפדפן וגשו לכתובת המוצגת בלוג (למשל `https://localhost:50169` או `http://localhost:50168`).
